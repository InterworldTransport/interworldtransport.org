38  /**
39   * This class contains cladosF numbers that act together as the coefficients of a monad. They are all children of 
40   * ProtoN and implement Field, so they have both a sense of 'units' and support basic arithmetic operations. Which
41   * numbers are contained internally, therefore, is tracked by two private elements. One contains a reference to a 
42   * Cardinal that all the numbers should share. The other is a reference two one of the CladosField elements so we
43   * know whether this Scale is expected to contain real or complex numbers and at what level of floating point 
44   * precision. Access to the two private elements is managed by their 'get' methods. getCardinal() and getMode(). 
45   * There are set methods for them too, but they are package protected methods that should not be handled much by 
46   * developers of physical models.
47   * <br><br>
48   * The data structure used to represent 'coefficients' used to be a fixed array that had the same length as the 
49   * number of blades in a monad's basis. That has been modernized to a TreeMap contained within this class. 
50   * The basis against which the map is applicable can be referenced by another private element, but shouldn't be 
51   * manipulated once set. The private element is finalized.
52   * <br><br>
53   * The TreeMap keys are Blades from the basis and reference equality is the required expectation when comparing them. 
54   * Typical use of keys occurs with streams that effectively iterate through the blades for access to coefficients in 
55   * the encompassing vector space. Information within a blade is far less important than which blade it is, thus 
56   * reference equality is what is needed.
57   * <br><br>
58   * Map Values are CladosF numbers like RealF or ComplexD. Because they are objects instead of primitives, they 
59   * behave much like Java's boxed primitives. In fact, they would BE those boxed primitives if not for the need to 
60   * track units in physical models. For example, one meter is not one second. No equality test should pass.
61   * <br><br>
62   * Because values are objects, care must be taken once one has a reference to them. Any reference to one enables a 
63   * developer to change it without the Scale or Monad knowing. This is the hydra monster named Mutability. It IS a 
64   * danger here. Many of Scale's methods copy inbound numbers to avoid altering them, but some do not INTENTIONALLY.
65   * <br><br>
66   * 1. Coefficient settors that accept arrays do NOT copy values before placing them in the internal map. BEWARE BEWARE
67   * <br><br>
68   * 2. Put() does not copy the incoming value before placing it in the internal map. Again... BEWARE.
69   * <br><br>
70   * 3. Coefficient settors that accept maps DO COPY values before placing them in the internal map. Any object 
71   * from which values are taken to be used here are safe from the hydra.
72   * <br><br>
73   * 4. All gettors for coefficients provide direct references to values in the map. The most common use is 
74   * INTENTIONAL MUTABILITY, so... BEWARE THE HYDRA. The safest way to use them is within streams / lambdas.
75   * <br><br>
76   * GENERAL NOTE | Many of the methods for Scale look a lot like Monad, so one can reasonably wonder why all the 
77   * extra stuff in Monad when Scale looks enough like a tuple to represent things. The primary difference is that Scale 
78   * contains only the coefficients and references a basis like what we got used to as students. That's not enough 
79   * because a basis is only enough to represent linear combinations for a vector space. Other geometric meanings aren't 
80   * in the basis. They are in the product table. Combining product table and basis into an 'algebra' gives a MUCH 
81   * better description of a 'tuple's' reference frame than a vector space.
82   * <br><br>
83   * 
84   * @version 2.0
85   * @author Dr Alfred W Differ
86   * @param <D> CladosF number like RealF, RealD, ComplexF, ComplexD. They must be
87   *            children of ProtoN AND implement Field.
88   */
89  public final class Scale<D extends ProtoN & Field & Normalizable> implements Unitized, Modal {

105 	/**
106 	 * This tree map is that actual list of weights mapped by their applicable blade. In use, one calls the 
107 	 * Scale's get(Blade) to get a generic that happens to be a CladosF.ProtoN child. One can also call a 
108 	 * number of specialized gettors to get weights for well named blades.
109 	 * <br><br>
110 	 * This feature used to be a simple array of particular children of ProtoN, but that made for several 
111 	 * different... and mostly related implementations of Scale or of burying Scale in Monad and maintaining 
112 	 * several mostly related versions of those. Using a map like this reduces the family of objects in CladosG 
113 	 * at the cost of swapping data structures from an array to a map.
114 	 * <br><br>
115 	 * This feature ALSO used to be a hash map (java's IdentityHashMap), but hash maps don't ensure the 
116 	 * extraction of values arrive in any particular order. That makes a mess of the design where streams are 
117 	 * used to deliver pieces of geometry or numbers to lambda functions. If the weights storied in this map 
118 	 * emerge in unpredictable ways, then all operations must act on blades AND numbers which we are trying to avoid. 
119 	 * Getting a predictable order (from a TreeMap) comes at a small performance cost that simply must be paid.
120 	 */
121 	protected TreeMap<Blade, D> map;

133 	 * This method streams through the weights and reports back true if they all self-report as being zero.
134 	 * <br><br>
135 	 * @param <D> stands in for a ProtoN child class
136 	 * @param pS Scale to be tested for having all zero weights
137 	 * @return boolean response that is true if all weights are zero and false otherwise.
138 	 */
139 	public final static <D extends ProtoN & Field & Normalizable> boolean isZero(Scale<D> pS) {
140 		return switch (pS.getMode()) {
141 			case COMPLEXD -> pS.weightsStream().allMatch(w -> ComplexD.isZero((ComplexD) w));
142 			case COMPLEXF -> pS.weightsStream().allMatch(w -> ComplexF.isZero((ComplexF) w));
143 			case REALD -> pS.weightsStream().allMatch(w -> RealD.isZero((RealD) w));
144 			case REALF -> pS.weightsStream().allMatch(w -> RealF.isZero((RealF) w));
145 		};
146 	}

148 	/**
149 	 * This method is for detecting cardinal differences in an array of numbers to be used as weights for a monad.
150 	 * The Scale might be able to tolerate them, but the monad won't when it is put to use. TypeMatching will fail.
151 	 * <br><br>
152 	 * return value +1: Incoming numbers perfectly match the standard cardinal.			Suggest using them.
153 	 * return value  0: Incoming numbers are a jumbled mess wrt the standard cardinal. 	Suggest tossing them.
154 	 * return value -1: Incoming numbers perfectly MISmatch the standard cardinal.		Suggest changing standard.
155 	 * <br><br>
156 	 * @param <D> 	stands in for a ProtoN child class
157 	 * @param pCard Cardinal to be used as the measure for deviations in the offered number array.
158 	 * @param pIn 	array of ProtoN children to be tested for nulls
159 	 * @return integer offered for decisions about keeping the numbers or the standard cardinal.
160 	 */
161 	public final static <D extends ProtoN & Field & Normalizable> int testCardinalMatchesIncoming(Cardinal pCard, D[] pIn) {
162 		long standardMatch = IntStream	.range(0, pIn.length)								
163 										.filter(i -> pIn[i].getCardinal() == pCard )
164 										.count();
165 		long internalMatch = IntStream	.range(1, pIn.length)								
166 										.filter(i -> pIn[i].getCardinal() == pIn[0].getCardinal())
167 										.count();				
168 
169 		//If standardMatch = pIn.length						: Cardinal match is perfect and the incoming numbers can be re-used with no cardinal resets.
170 		//If standardMatch is between {0, pIn.length}		: Cardinal match is a mess and the incoming numbers are not typeMatches for each other.
171 		//If standardMatch = 0								: Nothing matches and we might need to swap the standard cardinal
172 		//	If internalMatch = pIn.length -1				: Cardinals match perfectly within the incoming numbers and we should swap the standard.
173 		//	If internalMatch is between {0, pIn.length -1}	: Cardinal match is a mess
174 
175 		return 	(standardMatch == pIn.length) ? 1								//Standard Match is perfect
176 				: (standardMatch == 0 & internalMatch == pIn.length -1) ? -1 	//Standard Match is perfectly wrong... change the standard.
177 				: 0;															//Garbage offered. Toss it!
178 	}

181 	 * This method is for detecting mode difference in an array of numbers to be used as weights for a monad.
182 	 * The Scale might be able to tolerate them, but the monad won't... so they are rejected at Scale.
183 	 * <br><br>
184 	 * @param <D> 	stands in for a ProtoN child class
185 	 * @param pMode CladosField to be used as the measure for deviations in the offered number array.
186 	 * @param pIn 	array of ProtoN children to be tested for nulls
187 	 * @return boolean True if no mode mismatches are present in the array. False if any are.
188 	 */
189 	public final static <D extends ProtoN & Field & Normalizable> boolean validateModeIncoming(CladosField pMode, D[] pIn) {
190 
191 		return IntStream	.range(0, pIn.length)								
192 							.filter(i ->	!(pIn[i] instanceof RealF & pMode == REALF)
193 										& 	!(pIn[i] instanceof RealD & pMode == REALD)
194 										& 	!(pIn[i] instanceof ComplexF & pMode == COMPLEXF)
195 										& 	!(pIn[i] instanceof ComplexD & pMode == COMPLEXD) )
196 							.count() == 0;
197 	}

307 	/**
308 	 * This method 'conjugates' blades of the algebra, but leaves the numbers untouched.
309 	 * blades of the algebra untouched. 
310 	 * <br><br>
311 	 * These are Shrirokov's 'standard conjugates' that flip signs on blocks of weights
312 	 * in powers of two. For example...<br>
313 	 * invoke with (0) is the identity. It flips nothing.<br>
314 	 * invoke with (1) is the main involution. It flips every odd grade... so Integer.lowestOneBit(j)) == 1 <br>
315 	 * invoke with (2) is the reverse involution. It flips every other pair of grades... so Integer.lowestOneBit(j/2)) == 1 <br>
316 	 * invoke with (3) is unnamed, but flips every other quartet of grades... so Integer.lowestOneBit(j/4)) == 1 <br>
317 	 * invoke with (4) is unnamed, but flips every other octet of grades... so Integer.lowestOneBit(j/8)) == 1 <br>
318 	 * etc.
319 	 * <br><br>
320 	 * @param pWhich int used for pow(2, pWhich-1) which is the grade block size for sign switches.
321 	 * @return Scale after modification of numbers.
322 	 */
323 	public Scale<D> conjugateShirokov(int pWhich) {
324 		if (pWhich <1) 			return this;
325 
326 		int power = pow((byte) 2, pWhich-1).intValue();			//1 -> 2^0=1	2 -> 2^1=2
327 		gBasis.gradeStream().filter(j -> (Integer.lowestOneBit(j/power)) == 1).forEach(grade -> {
328 			gBasis.bladeOfGradeStream((byte) grade).forEach(blade -> {
329 				switch (mode) {
330 				case REALF:						//Tricky here. This case falls through to the next and gets handled.
331 				case COMPLEXF:
332 					(map.get(blade)).scale(CladosConstant.MINUS_ONE_F);
333 					break;						//Both cases handled in one then break.
334 				case REALD:						//Tricky here. This case falls through to the next and gets handled.
335 				case COMPLEXD:
336 					(map.get(blade)).scale(CladosConstant.MINUS_ONE_D);
337 				}			//Both cases handled in one then done.
338 			});
339 		});
340 		return this;
341 	}

388 	 * This method imitates one from V1.0 and should be rarely used. For old code
389 	 * needing to handle coefficients directly, this is how to get at them. The
390 	 * smarter approach, though, is to call the parent class method values() and
391 	 * receive a Collection of coefficients instead of an array. Do it that way and
392 	 * they are already of a known ProtoN child class.
393 	 * <br><br>
394 	 * Since the internal map can accept any of the CladosF numbers as values, there
395 	 * is a cast to a 'generic' type within this method. This would normally cause
396 	 * warnings by the compiler since the generic named in the internal map IS a
397 	 * ProtoN child AND casting an unchecked type could fail at runtime.
398 	 * <br><br>
399 	 * That won't happen here when CladosF builders are used. They can't build
400 	 * anything that is NOT a ProtoN child. They can't even build a
401 	 * ProtoN instance directly. Therefore, only children can arrive as the
402 	 * value parameter of the 'put' function. Thus, there is no danger of a failed
403 	 * cast operation... until someone creates a new ProtoN child class and
404 	 * fails to update all builders.
405 	 * <br><br>
406 	 * @return an array of ProtoN children.
407 	 */
408 	public D[] getNumbers() {
409 		switch (mode) {
410 			case REALF : return (D[]) map.values().toArray(RealF[]::new);
411 			case REALD : return (D[]) map.values().toArray(RealD[]::new);
412 			case COMPLEXF : return (D[]) map.values().toArray(ComplexF[]::new);
413 			case COMPLEXD : return (D[]) map.values().toArray(ComplexD[]::new);
414 			default : return null;
415 		}
416 	}

475 	 * asking the question regarding the value rather than handle all the various

477 	 * <br><br>
478 	 * NOTE this tends to get used in filters in streams to minimize the number of
479 	 * coefficients processed in arithmetic operations. Non-zero ones contribute
480 	 * non-zero results to products, so this especially matters in O(N^2)
481 	 * calculations.
482 	 * <br><br>

484 	 * @return boolean False if the related value evaluates as ZERO in whatever

488 		switch (mode) {
489 			case COMPLEXD :	return !ComplexD.isZero((ComplexD) map.get(pB));
490 			case COMPLEXF : return !ComplexF.isZero((ComplexF) map.get(pB));
491 			case REALD : return !RealD.isZero((RealD) map.get(pB));
492 			case REALF : return !RealF.isZero((RealF) map.get(pB));
493 			default : return true;
494 		}

584 			}

686 	 * This method normalizes the coefficients as if they were a vector in 2^N vector space described by the 
687 	 * implied basis from the monad. It's pretty simple, though. Just add up the squares of the numbers and 
688 	 * then take the square root to determine the magnitude and then invert that to scale the original numbers.
689 	 * <br><br>
690 	 * Since the internal map can accept any of the CladosF numbers as values, there is a cast to a 'generic' 
691 	 * type within this method. This would normally cause warnings by the compiler since the generic named in
692 	 *  the internal map IS a ProtoN child AND casting an unchecked type could fail at runtime.
693 	 * <br><br>
694 	 * That won't happen here when CladosF builders are used. They can't build anything that is NOT a ProtoN 
695 	 * child. They can't even build a ProtoN instance directly. Therefore, only children can arrive as the 
696 	 * value parameter of the 'put' function. Thus, there is no danger of a failed cast operation... until 
697 	 * someone creates a new ProtoN child class and fails to update all builders.
698 	 * <br><br>
699 	 * @throws FieldException 	This happens when normalizing something that has a zero magnitude. 
700 	 * 							The exception is thrown by the number's invert() and passed along.

702 	public void normalize() throws FieldException {
703 		this.scale((D) modulusSum().invert());

721 	 * This method scales all values in the internal map by the value offered provided there is no typeMatch failure. 
722 	 * When there IS a type mismatch the number simply does not get scaled.
723 	 * <br><br>
724 	 * The first stream filters for weights that pass the match test.
725 	 * The second stream scales them.
726 	 * That means the embedded IllegalARgumentException will never be thrown.
727 	 * <br><br>
728 	 * @param pIn ProtoN child to use as a scaling element. 
729 	 * 				Mode and cardinal MUST match values in map.
730 	 * @param <T> ProtoN child generic type support. Must also implement Field.
731 	 * @return Scale object. Just this object after modification.
732 	 */
733 	public <T extends ProtoN & Field & Normalizable> Scale<D> scale(T pIn) {
734 		if (	!(pIn instanceof RealF & getMode() == REALF)
735 			& 	!(pIn instanceof RealD & getMode() == REALD)
736 			& 	!(pIn instanceof ComplexF & getMode() == COMPLEXF)
737 			& 	!(pIn instanceof ComplexD & getMode() == COMPLEXD) )
738 			throw new IllegalArgumentException("Offered scaling number MUST mode match.");
739 
740 		if (pIn.getCardinal() == card & weightsStream().allMatch(x -> ProtoN.isTypeMatch(x, pIn))) {
741 			weightsParallelStream()
742 				.forEach(y -> {	try {y.multiply(pIn);} 
743 								catch (FieldBinaryException e) {
744 									throw new IllegalArgumentException("Can't scale with NaN or isInfinite.");
745 								}	//Field binary exception is also thrown for cardinal mismatches
746 									//but those were caught at the top of the conditional causing 
747 									//this method to do absolutely nothing. Anything caught down here 
748 									//should stop events the same way dividing by ZERO does.
749 			});
750 		}
751 		return this;
752 	}

858 	 * This coefficient settor accepts an array of ProtoN numbers, assumes they are in basis index order, and then 
859 	 * inserts them into the internal map by blade at that index.
860 	 * <br><br>
861 	 * NOTE | Do NOT use this method if you intend the offered coefficient array to be disconnected from this object. 
862 	 * IT WON'T BE!. If you really must use this method that way, copy your coefficients first.
863 	 * <br><br>
864 	 * NOTE | Do NOT try to change the mode for the scale either. Once set, mode is fixed.
865 	 * <br><br>
866 	 * @param <T> 	is a child of ProtoN used as the generic identity of the numbers.
867 	 * @param pIn 	Array of ProtoN children
868 	 * @return Scale after this object is modified.
869 	 * @throws IllegalArgumentException This happens if the offered array not suitable to cover the basis.
870 	 * 									All blades must be covered. NO NULL numbers.
871 	 */
872 	protected <T extends ProtoN & Field & Normalizable> Scale<D> setNumbers(T[] pIn) {
873 		if (pIn == null)																//Nulls aren't tolerated
874 					throw new IllegalArgumentException("Offered array of coefficients MUST NOT be null.");
875 		if (!Scale.validateNoNullsIncoming(pIn))										//Seriously! They aren't tolerated.
876 					throw new IllegalArgumentException("Offered array of coefficients MUST NOT contain nulls.");
877 		if(!Scale.validateModeIncoming(getMode(), pIn))									//Mixed mode isn't tolerated either.
878 					throw new IllegalArgumentException("Offered array of coefficients MUST mode match.");
879 		if (pIn.length != gBasis.getBladeCount())										//Offered array MUST cover the basis.
880 					throw new IllegalArgumentException("Offered array of coefficients MUST cover every blade in the basis.");
881 
882 		switch (Scale.testCardinalMatchesIncoming(card, pIn)) {
883 			case 0  : throw new IllegalArgumentException("Coefficients passed are a jumbled mess of cardinals.");
884 			case -1 : card = pIn[0].getCardinal();										//-1 case ALSO uses +1 action
885 			case +1 : gBasis.bladeStream().forEach(blade -> {
886 									setNumber(blade, pIn[gBasis.find(blade) - 1]);		//nulls checked again which is okay
887 																						//the top level check stops ALL mutation.
888 								});
889 			default : ;
890 		}
891 		return this;
892 	}

Clados Packages

Clados is a set of java packages implementing multi-vectors and multi-vector lists in Clifford Algebras.
cladosG Geometry
cladosF Numbers
cladosGExceptions Geometry related exceptions
cladosFExceptions Number related exceptions
cladosGTest JUnit tests for cladosG
cladosFTest JUnit tests for cladosF

If you don't know what a Clifford Algebra is, you are encouraged to read about them. Geometric Algebras are also Clifford Algebras and provide geometry modeling choice variations, so don't bypass content describing them either. They all offer a tidy mathematical formalism combining many disparate tools of geometry enabling one to wield them in algebraic ways.

The best way get started nowadays is to drop by the Bivector.net Discord Server. There is also a lot of You Tube content from recent years ranging from teaching material from sudgylacmoe to research grade reference from eigenchris, but many of the creators can be found on the Discord.

Project History

The tools contained in this library evolved from work done at UC Davis between the late 1970's and early 1990's. It is intended to support physical models that rely upon geometry in the style taught by Prof. Ken Greider.

The word 'Clados' is an acronym for CLifford Algebra Derived ObjectS. It was chosen to avoid math library name collisions and because it comes close to the concept of 'clade' which this author finds apropos for multivectors.

Basic and complete geometric objects are modeled by the Monad class which most people think of as 'Multivector.' A modeling requirement for aggregates of multivectors drove the main decision to use 'Monad' instead because 'Mult-multivector' is a mouthful.

Nyads are Monad aggregates and are modeled as lists. Lists of increasing size form a series {Monad, Dyad, Triad, Tetrad, ...}, so there is some sense to our naming convention. However, for implementors not interested in aggregates, the name 'Monad' is unfortunate. Please treat it as a synonym for multivector.

API

The primary use case for Clados is as a low level geometry library, so the API documentation is critical to its purpose. When used properly, a library user should never have to concern themselves with how Clifford algebras are represented. Intead they may focus on how geometry is represented and then folded into physical simulations.

The API docs show which classes are present and how to interact with them, but private elements and methods do most of the low level work where they are intentionally kept out of reach of most library users. One may look at the code directly, but one should never have to worry about how blade multiplication works especially when it comes to bit key handling.

Other References

There is a project related book that might help, but it is admittedly stale AND doesn't have introductory material. The book focuses upon package contents, what they do and why, and finishes with usage examples largely associated with physics simulations.

Two significant parts are very stale: the API and the Mechanics model. Use the API Docs (Javadocs) in the link above because the material in the book relates to Clados v1.0. The physics models won't survive unchanged when this library reaches v2.0 either and that's almost done. So until the book is updated, treat it as an exploration/research project regarding polycovariance and the broader meaning of 'meaning' in physical models.

Maintain a healthy sense of skepticism... and try things for yourself.

Usage

Clados is a single java module with four packages meant to be used as a math library. Place the clados jar where you normally place 3rd party libraries and adjust your classpath accordingly.

Don't have the jar file? Build it from source. The project has been through several cycles of different build tools, but is currently set up for Maven. I'll release the v2.0 jar soon, but a current build by you will have almost everything in it.
